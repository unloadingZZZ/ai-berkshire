# AI Berkshire 股票筛选智能体：规则全解

> 分析日期：2026-07-10
> 说明：本项目没有单一命名为"股票筛选智能体"的模块，而是由 **3个基于LLM+WebSearch的筛选Skill** 和 **2个基于API/爬虫的量化脚本** 构成的筛选体系。它们分工不同、可独立使用也可串联成完整漏斗。本文档逐一拆解各自的筛选规则、判定阈值与输出格式。

**两条筛选路径的本质区别**：

| 维度 | 质性筛选（Skill） | 量化筛选（Python脚本） |
|------|------------------|----------------------|
| 执行者 | Claude + WebSearch，人工可读规则 | 脚本自动跑，规则写死在代码里 |
| 输入 | 公司名/行业/主题（自然语言） | Ticker 列表 / watchlist.json |
| 输出 | Markdown 报告，逐条留痕 | 终端打印 + JSON/CSV 数据文件 |
| 更新方式 | 每次运行重新搜索 | 需要手工 `--update` 补基本面数据 |

## 二、quality-screen（去劣筛选）— 质性筛选的地基

**定位**：不挑好公司，只排除"确定不好"的公司。宁可漏网，不可误杀。

### 7条硬指标（触犯任一条即排除，除非命中豁免）

| # | 指标 | 排除阈值 | 衡量什么 |
|---|------|---------|---------|
| 1 | 10年平均ROE | < 8% | 资本效率 |
| 2 | 5年累计自由现金流 | 为负 | 利润是否纸面富贵 |
| 3 | 利息覆盖倍数（EBIT/利息） | < 2倍 | 偿债安全（银行/保险不适用） |
| 4 | 长期毛利率 | < 15% | 定价权 |
| 5 | 经营现金流/净利润（5年均值） | < 0.7 | 利润质量 |
| 6 | 长期净利率 | < 5% | 抗风险能力 |
| 7 | 5年总股本膨胀 | > 20%（非并购） | 股东是否被稀释 |

### 3条豁免规则

| 豁免 | 针对 | 触发条件（需同时满足） | 典型案例 |
|------|------|----------------------|---------|
| A：战略投入期 | 第1条ROE | 上市<10年 + 毛利率>30% + 近2年OCF为正 | 美团 |
| B：主动低利润率 | 第6条净利率 | 毛利率>30% + 近2年净利率回升至5%+或明确上升 | 亚马逊 |
| C：高周转薄利 | 第4、6条 | ROE>20% + OCF/净利>1.0 + 会员制/佣金/高周转模式 | Costco |

### 执行流程
输入支持个股/行业/市场指数/主题四种模式 → 批量模式先WebSearch确定10-30家公司清单 → 并行Agent逐家收集7项数据 → 逐条判定✅/❌/⚠️ → 输出汇总表+通过名单+排除名单+豁免名单+（行业模式）通过率与分层总结。

### 适用例外
- 银行/保险：跳过第3条（利差经营模式）
- REIT：用"核心营运利润ROE"代替普通ROE
- 周期性行业：用覆盖一个完整周期的均值，非单年数据
- 数据缺失：标注"数据不足"，不直接判定通过/不通过

### 局限性（Skill自述）
通过筛选 ≠ "确定好"，只是排除了"确定不好"。仍需进一步评估商业模式可持续性、管理层可信度、当前估值、竞争格局变化——去劣是第一步，不是最后一步。

## 三、industry-funnel（行业漏斗筛选）— 从全市场到3家

**定位**：给定一个行业/方向，用统一标准逐层过滤，每层都留淘汰理由，不黑箱。与 `industry-research`（产业链全景）互补，一个看结构，一个选个股。

### 四层漏斗

```
第一层：全市场扫描       30-60家   活跃度(A)+涨幅(B)+市值(C) 三类并集
   ↓ 5条硬指标
第二层：粗筛            ≤10家    5条全过 / 4条过+1条接近
   ↓ 结构化分析
第三层：精细分析         ≤10家    每家300-500字
   ↓ 组合互补性选择（非纯打分排序）
第四层：四大师深度分析    3家     每家800-1200字
```

### 第一层扫描池定义（A∪B∪C）

| 类别 | 定义 |
|------|------|
| A 成交活跃度 | 近30天日均成交额行业前30（A股/港股/美股各取） |
| B 涨幅榜 | 近30天涨幅前20 ∪ 近90天涨幅前20 |
| C 市值锚定 | 行业内市值前30（无论涨跌） |

必须覆盖 A股/港股/美股/日韩台欧 + 未上市IPO候选，不可因资料少而漏掉非英文市场标的。

### 第二层5条硬指标

| # | 指标 | 通过标准 | 放宽条件 |
|---|------|---------|---------|
| 1 | PE估值 | 合理（历史区间/同业对比） | 高成长可放宽至 PEG<1.5 |
| 2 | ROE | >15% 或近3年趋势改善 | 重资产行业可放宽 |
| 3 | 经营现金流 | 为正且占净利润>70% | — |
| 4 | 资产负债率 | <60% | 公用事业/电力放宽至70% |
| 5 | 护城河快评 | ★★★以上（品牌/转换成本/网络效应/规模效应/技术牌照5类之一） | — |

**保留规则**：5条全过→保留；4条过+1条接近→保留标黄；不足4条→淘汰注明理由。若保留超12家，护城河门槛提到★★★★重筛一次。

### 第三层终选3家的选择逻辑
不是打分排序取前3，而是按组合互补性：至少1家"高确定性低弹性"（巴菲特型）+ 1家"中等确定性中等弹性"（成长型）+ 可选1家"高弹性高风险"（期权型）。凑不齐3家时宁可写"终选2家+1家观察"。

### 第四层四大师视角
段永平（生意本质/本分）→巴菲特（护城河5类打分+10年后是否还在+安全边际）→芒格（3条最可能失败路径+最坏情景估值+反向论证）→李录（文明级趋势 vs 阶段性热潮+终局判断）→综合推荐度（★评级+仓位类型+建议买入区间）。

### AI偏见自觉清单（该Skill特有）

| 偏见 | 表现 | 应对 |
|------|------|------|
| 龙头偏好 | 大市值公司资料多、篇幅长看起来更好 | 按硬指标和护城河打分，不按篇幅排序 |
| 英文偏好 | 美股资料丰富，A/H股被低估 | 中英文都搜 |
| 故事偏好 | 高涨幅+媒体热度被误判为好公司 | 区分"AI收入占比"与"AI故事占比" |
| 当下偏好 | 只看当前财务好的公司 | 允许"趋势改善"作为放宽条件 |
| 上市偏好 | 漏掉赛道最好的未上市玩家 | 必须列"未来IPO候选" |

### 准出流程（发布前必做）
写完报告后用 `tools/report_audit.py extract` 抽取15%随机抽样清单 → 人工/Agent核实数据 → `tools/report_audit.py verdict` 输出【准出】/【打回】判决，全部通过才能发布。

## 四、bottleneck-hunter（供应链瓶颈猎手）— 反向选股

**定位**：不问"AI推荐什么股票"，问"如果趋势继续扩张，哪一环会先不够用"。从物理供应链的咽喉位置反推标的，聚焦第二、三层被低估值的环节（第一层如GPU/HBM已被充分定价）。

### 第一步：超级趋势确认（4条全部满足才追踪）

| 标准 | 要求 |
|------|------|
| 持续性 | 至少3-5年确定性增长 |
| 物理性 | 需要实际硬件/材料/设备建设，非软件升级 |
| 规模性 | 全球资本开支 > 500亿美元/年 |
| 加速性 | 需求增速 > 供给扩产速度 |

### 第二步：供应链分层拆解

```
Layer 0 终端 → Layer 1 核心组件（已充分定价，alpha有限）
           → Layer 2 子组件/材料（重点扫描区，alpha集中）
           → Layer 3 上游设备/原料 → Layer 4 基础设施（电力/冷却/土地）
```

### 第三步：瓶颈判定6条标准（每条🔴/🟡/🟢打分）

| # | 标准 | 🔴 | 🟡 | 🟢 |
|---|------|-----|-----|-----|
| 1 | 供给集中度 | 全球≤2家 | 3-5家 | >5家 |
| 2 | 扩产周期 | >2年 | 1-2年 | <1年 |
| 3 | 替代难度 | 不可替代 | 部分可替代 | 易替代 |
| 4 | 产能利用率 | >90% | 70-90% | <70% |
| 5 | 需求增速 | >50%/年 | 20-50% | <20% |
| 6 | 客户验证周期 | >1年 | 6-12月 | <6月 |

**瓶颈评级**：🔴≥4个→S级（单点故障，最高优先级）；3个→A级；1-2个→B级；无🔴→非瓶颈跳过。

### 第四步：估值硬门槛（不可被瓶颈纯正度或叙事覆盖）

**估值红灯**（任一命中 → 信号强度封顶★★，标注"估值透支"）：
1. 市值 > TAM的20%
2. PS>30x 且收入增速<100%（增速>100%可豁免但需标注需持续验证）
3. 市值 > 5年乐观收入预测的10倍
4. 增发后60天内股价翻倍（情绪驱动）

**估值黄灯**（需额外解释，否则降级）：亏损+PS>15x；PS是同行盈利公司5倍以上；PE>80x需算PEG。

**估值绿灯**（加分）：PS<10x且增长中；PE<30x且有护城河。

**安全边际检验（必做）**：以当前市值买入，最乐观情景全部兑现，10年后25x PE退出，年化回报是多少？<10% → 标注"不具备安全边际"。

### 初筛4条快速过滤

| 标准 | 要求 |
|------|------|
| 上市状态 | 已上市（A/港/美/日/台/欧） |
| 瓶颈业务占比 | >30%收入来自瓶颈环节 |
| 市值 | 优先<100亿美元（大市值已被充分定价） |
| 流动性 | 日均成交额>100万美元 |

### 第五步：交叉验证（正向+反向）
正向：客户验证/收入验证/价格验证/产能验证/资本验证。
反向（芒格式）：聪明人为什么不买？瓶颈能否被绕过？中国/其他玩家能否快速复制产能？需求放缓50%会怎样？管理层是否高点增发过？

### 信号强度评级（估值直接决定上限）
★★★★★ 多重验证+估值绿灯 → ★★★★ 大部分验证+绿/黄灯 → ★★★ 逻辑成立+黄灯可接受 → ★★ 早期信号或估值红灯封顶 → ★ 纯概念未验证。

### 每小时定时扫描模式
只在"有料"时出报告：新闻扫描（关键词：bottleneck/shortage/涨价/缺货）→ 市场信号（异常波动>5%）→ 财报公告检查 → 估值机会检查 → 无新发现则仅记日志不出报告。文件命名 `HH-MM-标的代码.md`（有标的）或 `HH-MM-信号扫描.md`（无标的）。

### 核心原则（该Skill置顶）
1. 不让AI推荐股票，让AI拆供应链——问题比答案重要
2. 只关注需要实际物理产品/材料/设备的环节
3. 聚焦第二、三层，不追已充分定价的龙头
4. 每个结论至少2个独立信源
5. 瓶颈有时效性，需判断解除时间窗
6. 小市值≠好机会，仍需过财务质量关
7. **瓶颈真实≠投资机会**，估值是硬门槛，不可被叙事覆盖

## 五、stock_screener.py（动量+价值 量化筛选）

**定位**：技术面动量发现 + 基本面价值验证的二层筛选，产出分级仓位信号，跑在本地脚本，非LLM驱动。

### 第一层：动量发现（技术面，硬编码规则）

| 条件 | 计算方式 | 阈值 |
|------|---------|------|
| 60日新高 | 收盘价 > 过去60日最高价 | 触发其一 |
| 近5日突破 | 近5个交易日内任一日收盘价创60日新高 | 触发其一 |
| 放量确认 | 近5日均量 / 近20日均量 | >1.5 |

`triggered = (60日新高 或 近5日突破) 且 放量`，两个条件都必须满足才进入待选池。

### 第二层：价值验证（6维评分）

| # | 维度 | 判定条件 |
|---|------|---------|
| 1 | 营收加速 | 本季同比增速 > 上季同比增速（无上季数据则要求 >20%） |
| 2 | 毛利率扩张 | 本季毛利率 > 上季（或 >55%）（无历史则要求>45%） |
| 3 | 盈利惊喜 | EPS超预期 > 10% |
| 4 | 营收高增长 | 同比增速 > 15% |
| 5 | 毛利率健康 | 毛利率 > 40% |
| 6 | 毛利连续改善 | 连续2季毛利率环比提升（数据不足则退化为单季对比） |

`score = 6项中满足的数量`

### 独立通过条件（不看6维总分，单条即可直接过）
- 条件A：毛利率连续2季改善 且 毛利率>45%（解决NVDA 2023-01型漏判）
- 条件B：EPS超预期>30%（解决周期股底部信号，如MU）

### 信号分级与仓位建议

| 条件 | 评级 | 建议仓位 |
|------|------|---------|
| score≥5，或(score≥4 且独立通过) | BUY_8% 确信仓 | 8% |
| score≥4，或(score≥3 且独立通过) | BUY_5% 标准仓 | 5% |
| score≥3 | BUY_3% 试探仓 | 3% |
| 仅独立通过（score<3） | BUY_3% | 3% |
| 动量未触发 | SKIP | — |
| 动量触发但无基本面数据 | WATCH | 需 `--update` 补数据 |
| 动量触发+有数据但score不足且不满足独立条件 | PASS | 继续观察 |

### 数据依赖
价格数据用curl拉取Yahoo Finance日线（绕过Python SSL问题）；基本面数据（营收同比/毛利率/EPS超预期）需手工通过 `--update TICKER` 交互录入，存于 `data/fundamentals.json`；watchlist 分组存于 `data/watchlist.json`（默认覆盖美股AI芯片/应用/基建、加密货币、港股互联网几组，A股留空待扩展）。

## 六、morningstar_fair_value.py（晨星公允价值量化筛选）

**定位**：批量抓取晨星筛选器API的公允价值估计，按潜在涨幅排序输出全市场Top100，是"晨星估值筛选"和"晨星深度低估"两个报告集的数据源。

### 抓取规则
- 数据源：Morningstar Screener API（`security/screener` 接口），限定美股（XNAS/XNYS）
- 抓取字段：SecId/Name/收盘价/星级(StarRatingM255)/量化公允价值/公允价值估计/公允价值不确定性/护城河(EconomicMoat)/行业/板块
- 过滤条件：仅保留 `FairValueEstimate:notnull`（有公允价值估计的标的）
- 分页抓取全部结果（每页100条），分页间隔0.3秒避免触发限流

### 计算与排序
```
潜在涨幅% = (公允价值 - 现价) / 现价 × 100
```
按潜在涨幅降序排序，输出Top100；同时保存全量数据到 `data/morningstar_fair_value_{日期}.csv`。

### 统计摘要（脚本自动输出）
- 低估股票占比（潜在涨幅>0）及其平均潜在涨幅
- 高估股票占比（潜在涨幅<0）
- **宽护城河(Wide Moat) + 低估** 的交集数量——这是后续"晨星深度低估"报告集重点深挖的候选池

### 与人工报告的关系
`reports/晨星估值筛选/` 下的报告是对该脚本输出的Top名单做的行业/主题重排（如"AI五层蛋糕50家公司公允价值排名"）；`reports/晨星深度低估/` 下每家公司一个文件夹，是对高潜在涨幅+护城河标的做的单公司深度研究（文件夹命名格式：`{公司名}-{护城河等级}-低估{百分比}%`）。

## 七、贯穿所有筛选规则的共性原则

这些原则写在 CLAUDE.md 或各Skill文件中，对所有筛选流程强制生效：

| 原则 | 具体要求 |
|------|---------|
| 客观性 | 不预设看多/看空，先摆数据→推逻辑→出结论；禁用"我认为""显然"等主观表述 |
| 正反两面 | 每个核心判断必须附反面论据 |
| 不黑箱 | 每一层被淘汰的标的必须留名字+理由，不能只留通过名单 |
| 估值是硬门槛 | 瓶颈猎手/漏斗筛选均明确：叙事再动人、瓶颈再真实，估值透支就封顶评级，不可覆盖 |
| 数据不足即如实说 | 标注"数据不足"/"估计"，不用推测填充确定性 |
| 反AI偏见 | 龙头偏好、英文偏好、故事偏好、上市偏好——industry-funnel 和 bottleneck-hunter 都专门列出对应清单 |
| 交叉验证 | 关键数据至少2个独立信源 |
| 银行/保险/REIT 例外处理 | 利息覆盖率、ROE等指标在特殊行业需替换或跳过 |

## 八、典型组合使用路径

```
路径A（行业驱动）：
industry-research（看清产业链格局）
   → industry-funnel（全市场漏斗精选3家）
   → investment-team / investment-checklist（单公司深挖）

路径B（去劣建池，本项目已实践）：
quality-screen（对100+候选公司批量去劣）
   → 沉淀为 筛选公司/{市场}召回池/README.md + 分行业清单
   → 对通过公司逐一 investment-team 深度研究

路径C（趋势反推）：
bottleneck-hunter（从超级趋势拆到瓶颈环节再到标的）
   → 估值硬门槛过滤 → 观察名单/深入研究分流

路径D（量化扫描，脚本化，可定时跑）：
stock_screener.py（动量触发 + 基本面打分）→ 分级仓位信号
morningstar_fair_value.py（公允价值排序）→ 宽护城河+低估交集 → 晨星深度低估单公司研究
```

**产出落地位置**：
- 质性筛选报告 → `reports/` 按 CLAUDE.md 命名规范存放
- 去劣筛选建池结果 → `筛选公司/{市场}召回池/`
- 瓶颈地图 → `reports/bottleneck-map/`
- 量化脚本输出 → `data/`（JSON/CSV，非Markdown报告）
